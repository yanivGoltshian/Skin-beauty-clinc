# מדריך ניהול האתר — סקין ביוטי קליניק

מערכת הניהול נמצאת בכתובת **`/admin`** של האתר:

> https://yanivgoltshian.github.io/Skin-beauty-clinc/admin/

היא מיועדת לבעלת העסק בלבד. דרכה אפשר לערוך את כל התוכן של דף הבית, את
הטיפולים, הקטגוריות ופרטי הקשר — בלי לגעת בקוד. כל שמירה מתפרסמת לאתר החי
תוך **1–3 דקות**.

---

## התחברות — פעם אחת

יש שתי שכבות כניסה:

1. **קוד כניסה** — קוד קצר וידידותי. ברירת המחדל: `skin2025`.
2. **טוקן GitHub** — מפתח אישי שמאפשר לשמור שינויים אל האתר. צריך ליצור אותו
   פעם אחת ולהדביק. הוא נשמר רק בדפדפן שלך (ב‑`localStorage`) ולא נשלח לאף
   מקום אחר חוץ מ‑GitHub.

### יצירת טוקן GitHub (פעם אחת, ~2 דקות)

1. היכנסי ל‑ https://github.com/settings/personal-access-tokens/new
   (Settings → Developer settings → **Fine‑grained tokens** → *Generate new token*).
2. **Token name**: משהו שתזהי, למשל `skin-admin`.
3. **Expiration**: בחרי תוקף (למשל 90 יום; אפשר לחדש בהמשך).
4. **Resource owner**: `yanivGoltshian`.
5. **Repository access** → *Only select repositories* → בחרי את
   **`Skin-beauty-clinc`** בלבד.
6. **Permissions** → *Repository permissions* → **Contents** → בחרי
   **Read and write**. (זו ההרשאה היחידה שצריך.)
7. לחצי **Generate token** והעתיקי את הטוקן — הוא מתחיל ב‑`github_pat_…`.
   ⚠️ הוא מוצג פעם אחת בלבד, שמרי אותו במקום בטוח.

### הדבקת הטוקן

1. פתחי את `/admin`, הזיני את קוד הכניסה (`skin2025`).
2. במסך «חיבור ל‑GitHub» הדביקי את הטוקן ולחצי **התחברי**.
3. זהו — מעכשיו את במסך העריכה. הטוקן זכור בדפדפן, לא צריך להדביק שוב.

> אם החלפת מחשב/דפדפן — פשוט חזרי על הדבקת הטוקן.

## מסך העריכה

בראש המסך יש ארבע לשוניות, בדיוק לפי מבנה האתר:

| לשונית | מה עורכים |
| --- | --- |
| **דף הבית** | וידאו וכותרת ההירו, פס ההכרזה, יתרונות, טקסט פתיחה + גלריה, מבצעים חמים, סטטיסטיקות, אבחון, טיפולים מומלצים, באנר מותג, פסיפס תמונות, «קצת עלינו». |
| **טיפולים** | רשימת הטיפולים — הוספה, עריכה ומחיקה (שם, תיאור, קטגוריות, תמונה, מאפיינים). |
| **קטגוריות** | ארבע קבוצות הטיפולים (שם, תיאור, צבע). |
| **הגדרות** | שם העסק, סלוגן, טלפון, וואטסאפ, כתובת, שעות, רשתות חברתיות, תגי אמון. |

כל אזור מוצג כ«אקורדיון» עם **תצוגה מקדימה** לצידו, כדי שתראי בדיוק איך זה
ייראה באתר.

### שמירה

- בכל לשונית יש בתחתית פס **«שמור ופרסם»**.
- אחרי שמירה יופיע «מתפרסם… (עד 3 דקות)». המתיני דקה‑שתיים ורענני את האתר.
- אם התחרטת לפני שמירה — «בטל שינויים» מחזיר את המצב האחרון שנשמר.
- המערכת שומרת **טיוטה אוטומטית** בדפדפן, כך שרענון בטעות לא מאבד עבודה.

### תמונות

- בהעלאת תמונה המערכת **מקטינה ומייעלת** אותה אוטומטית (כדי שהאתר יישאר מהיר).
- בתמונות מרכזיות יש **בורר מיקוד** — גררי את הנקודה כדי לבחור איזה חלק של
  התמונה יישאר במרכז החיתוך.
- קבצים «קבועים» (וידאו ההירו, פוסטר, פסיפס הגלריה, תמונת האבחון) מוחלפים
  במקום — פשוט העלי קובץ חדש והוא יחליף את הישן.

## איך זה עובד מאחורי הקלעים

האתר הוא **סטטי** ומתארח בחינם ב‑GitHub Pages. אין שרת. לכן השמירה עובדת כך:
מערכת הניהול רצה כולה בדפדפן, וכשלוחצים «שמור» היא כותבת את קובץ התוכן
ישירות ל‑GitHub (עם הטוקן שלך). כל כתיבה מפעילה בנייה מחדש של האתר, ולכן
השינוי עולה תוך 1–3 דקות.

### הערת אבטחה (חשוב)

זהו כלי לבעלת עסק **יחידה**. הטוקן נשמר בדפדפן שלך בלבד, ומוגבל להרשאת כתיבת
תוכן על מאגר האתר בלבד (Contents: Read and write). אל תשתפי את הטוקן ואל
תשתמשי במחשב ציבורי. אם חושדים שהטוקן דלף — מחקי אותו ב‑GitHub
(Settings → Developer settings → Tokens → Revoke) וצרי חדש.

## עריכה מקומית (למפתחים בלבד)

ל‑edit loop מהיר בלי לפרסם לאתר:

```bash
npm run admin
# פותח שרת פיתוח + API מקומי לכתיבה
# גשי אל  http://localhost:8787/admin/
```

במצב זה השמירות נכתבות אל קבצי ה‑JSON המקומיים (`src/data/*.json`) ולתיקיית
`public/`, ללא צורך בטוקן. מסך הניהול מזהה אוטומטית שהוא רץ מקומית ומציג תג
«מצב מקומי».

מבנה הנתונים:

- `src/data/site.json` — פרטי העסק והקשר.
- `src/data/homepage.json` — כל תוכן דף הבית.
- `src/data/products.json` — הטיפולים.
- `src/data/categories.json` — הקטגוריות.
- `public/images/**` — התמונות.
